# Artworksの画像管理方法について
## 現状：
- サムネイルと作品本体が別の画像になっている。
- 現在の画像圧縮スクリプトでは、サムネイル用画像は作品本体を単純に縮小するものになっている
- 画像はNext.jsの `<image>` タグで貼られていて、クライアントでの表示サイズに自動最適化される。

## 蓮田が思ったこと。
- サムネイルと作品本体でわざわざ別の画像をリポジトリに保存する意味はないのではないか
- Artworksの画像を作品本体のみ利用する形に変更できるなら、下に示すディレクトリ構造のように `images/artworks/--` に直接作品画像を収めれば良く、作品番号ごとにフォルダ分けする必要がなくなる。
```
images/
    ├── artworks/
            ├──089-1s.jpg
            ├──094-1s.jpg

```

あと、 `hasuda-portfolio\scripts\compress-images.ts` でオリジナルの画像を縮小するのと同時にfrontmatter設定済み（`number`, `imagePath`, `thumbnailPath`みたいな自明なところだけ）のmdファイルも生成できたら良いなって
